# Takia — Прототип приложения

Демо-прототип цифрового спутника паломника. HTML + JS, без бэкенда.

## Структура

```
takia_app/
├── index.html        — лендинг (главная страница)
├── app.html          — карта с точками
├── place.html        — страница точки
├── guides.html       — список руководств
├── guide.html        — шаг руководства (Умра/Хадж)
├── offline.html      — управление офлайн-данными
├── favorites.html    — избранное
└── assets/
    ├── data.js       — все данные (точки, руководства, пользователь)
    └── styles.css    — стили приложения
```

## Как опубликовать на GitHub Pages

1. Создай репозиторий на GitHub
2. Загрузи туда содержимое папки `takia_app/` (не саму папку, а файлы внутри)
3. Settings → Pages → Source: ветка `main`, папка `/ (root)`
4. Через минуту сайт будет на `https://ТВОЙ-ЛОГИН.github.io/ИМЯ-РЕПО/`
5. Открой `index.html` — карта загрузится автоматически (есть интернет → настоящая карта)

## Как опубликовать на ps.kz (или любом хостинге)

1. Зайди в файловый менеджер хостинга или подключись по FTP
2. Загрузи все файлы из `takia_app/` в `public_html/` (или корень сайта)
3. Открой `https://твойдомен.kz/` — должен открыться `index.html`
4. Карта работает сразу, потому что на хостинге всегда есть интернет

## Как работает карта

Карта загружает библиотеку **Leaflet** и тайлы карты (CARTO). Логика:

1. **Есть интернет (хостинг, Git Pages):** загружается настоящая
   детальная карта улиц — как в 2ГИС. Это основной режим.
2. **CDN недоступен:** код автоматически пробует 3 разных CDN
   (jsDelivr → unpkg → cdnjs) по очереди.
3. **Совсем нет интернета (например, файл открыт локально):**
   рисуется встроенная SVG-схема — страховка, чтобы прототип
   всегда что-то показывал. На реальном хостинге этот режим
   не используется.

**Вывод:** на Git/хостинге всегда грузится настоящая карта.
SVG-схема — только если открыть файл двойным кликом с диска
(`file://`), что для финального продукта неактуально.

## (Опционально) Встроить Leaflet в репозиторий

Для максимальной надёжности и скорости можно не зависеть от
внешних CDN, а положить Leaflet прямо в репозиторий:

1. Скачай два файла:
   - https://cdn.jsdelivr.net/npm/[email protected]/dist/leaflet.js
   - https://cdn.jsdelivr.net/npm/[email protected]/dist/leaflet.css
2. Положи их в папку `assets/`
3. В `app.html` найди массив `var sources = [...]` (около строки 230)
   и добавь первым элементом: `'assets/leaflet.js',`
4. В `app.html` и `place.html` замени CSS-ссылку Leaflet на:
   `<link rel="stylesheet" href="assets/leaflet.css" />`

После этого карта-движок грузится с твоего домена — быстро
и без зависимости от чужих серверов. Тайлы (картинки карты)
всё равно идут из интернета (CARTO/OSM) — это нормально,
в production их кеширует Service Worker (Модуль III).

## Что в прототипе работает

- Карта Мекки/Медины с точками всех категорий (25 реальных мест)
- Фильтры по категориям + по кухням (для еды)
- Поиск мест
- Страница точки: описание, маршрут, звонок, избранное
- Руководства Умра (4 этапа) и Хадж (5 этапов) с дуа и аудио
- Офлайн-страница с имитацией скачивания пакетов
- Избранное (хранится в браузере)
- Бургер-меню на лендинге для мобильных

## Что НЕ работает (нужен бэкенд)

- Реальная регистрация/вход (имитация — «вы Айгуль»)
- Реальные платежи Premium
- Реальное сохранение в БД
- Service Worker для настоящего офлайна
- Реальное скачивание офлайн-карт

Это следующий этап разработки с бэкендом (PHP/.NET).
